# AidUp Backend: Fixing Campaign Image & Banner Sender

This guide provides steps to fix how the backend handles campaign images and the banner in the `addCampain` process. Currently, the backend expects `banner` as a simple string in `req.body` and handles `images` as an uploaded file array. 

There are two primary ways to fix this depending on how the Android App sends the data:

## Option A: Using the first uploaded image as the banner (Recommended)

If the Android app uploads all campaign photos under the `images` field in a single multipart request, the easiest fix is to just take the first uploaded image and set it as the banner automatically.

### 1. Update `addCampain` in `controllers/campainController.js`
Modify the destructuring to pull `banner` out of the requirements from `req.body`, and assign it from `req.files`:

```javascript
// Remove 'banner' from req.body destructuring
const { title, description, category, goal_amount, goal, story, paiment_methods, organizer_id, goal_date } = req.body;

// Get image URLs that were generated by processMultipleImages middleware
const uploadedImages = req.files.images || [];
const uploadedVideos = req.files.videos || [];

if(uploadedImages.length === 0) {
    return res.status(400).json({ success: false, message: 'At least one image is required' });
}

// Automatically use the first image as the banner
const banner = uploadedImages[0];

// The rest of the images go to the 'images' array 
// (or you can keep all of them in the array if you prefer)
const images = uploadedImages; 
const videos = uploadedVideos;

// ... proceed to Campain.create({...})
```

---

## Option B: Uploading the banner as a separate precise file

If the Android app sends the banner as its own specific file part named `banner`, you must update the upload middleware and the controller.

### 1. Update Routes `routes/campainRoutes.js`
Add `banner` to the `upload.fields` array so Multer knows to expect it as a file:

```javascript
router.post(
    '/managecampain/add', 
    upload.fields([
        { name: 'banner', maxCount: 1 },    // <-- ADD THIS
        { name: 'images', maxCount: 10 }, 
        { name: 'videos', maxCount: 10 }
    ]),
    authorize('organizer'), 
    processImage, // Make sure processMultipleImages handles ALL these keys!
    uploadErrorHandler,
    auditLog('Create Campaign by organizer'), 
    addCampain
);
```

### 2. Update `addCampain` in `controllers/campainController.js`
Read the URL for the banner and the array of URLs for images from `req.files`:

```javascript
// Extract text fields
const { title, description, category, goal_amount, goal, story, paiment_methods, organizer_id, goal_date } = req.body;

// Extract file URLs
const banner = req.files.banner && req.files.banner.length > 0 ? req.files.banner[0] : null;
const images = req.files.images || [];
const videos = req.files.videos || [];

if(!banner) {
    return res.status(400).json({ success: false, message: 'Banner image is required' });
}

// ... proceed to Campain.create({...})
```

### Summary
The middleware `processMultipleImages` correctly resizes and transforms uploaded files into `.webp` paths and overwrites `req.files` to be arrays containing those URL strings. However, if your controller attempts to read `images` or `banner` out of `req.body`, it will fail because file data exists in `req.files`, not `req.body`. Ensure your controller references `req.files`!
